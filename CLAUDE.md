# Proc App вЂ” Claude Context

## What this project is
"PL Spectrum Stitcher / Converter" вЂ” a Windows PyQt5 desktop app for processing photoluminescence (PL) spectroscopy data during Benjamin Haubmann's PhD at TUM. It stitches multi-file power-series spectra into a single spectrum, converts between file formats (.origin / HDF5), visualizes spectra, and runs peak-fitting / threshold-detection analysis.

## Key facts
- **Entry point**: `stitchcraft.py` вЂ” builds the `QApplication`, applies the `qt_material` `dark_teal.xml` stylesheet, calls `plotting.apply_pg_theme()`, then shows `StitchApp`.
- **Main window**: `app.py` вЂ” `StitchApp(QMainWindow)`, a `QTabWidget` hosting the 6 tabs below.
- **Platform**: Windows-only PyQt5 app (no PySide/Qt6 constraint otherwise).
- **GitHub**: https://github.com/ge23jud/Proc.git
- **Dependencies**: pinned in `requirements.txt` (PyQt5, pyqtgraph, numpy, h5py, scipy, pandas, qt-material, opencv-python, Pillow, matplotlib вЂ” matplotlib is only needed because `nw_analysis.py` and `pl.py` import it at module level; see the `plotting/` note below). No `pyproject.toml`/`setup.py`.

## Tabs (`tabs/`)
- **`stitch.py`** вЂ” "Stitch / Convert": load `.origin` power-series files, preview spectra, pick per-pair overlap "transition spans" (draggable region), dark-spectrum subtraction, power calibration, save as HDF5 or `.origin`.
- **`visualizer.py`** вЂ” "Visualizer": load HDF5 files, overlay every power step's PL spectrum vs. energy (eV) across all loaded files, colored by file or by power; legend quantity is switchable (power / power density / pump fluence). No axis configuration (energy-only x-axis, always linear y, no per-power-step filtering вЂ” deliberately removed 2026-07 to simplify the tab) and no interactive power-step selection; a "Show Data" button opens a dialog listing every dataset/group/attribute in the selected file's HDF5 structure (via `io_utils._h5_contents_summary()`).
- **`analysis.py`** вЂ” "Analysis": peak-picking в†’ fit-window selection в†’ `fit_nw`/`integrate_spectra` (via `nw_analysis.py`) в†’ threshold detection (drag-select ASE region, linear fit, threshold В± error) в†’ L-L curve plot в†’ save results into the HDF5's `analysis` group. A click-driven wizard/state machine (`self._ana_mode`).
- **`spotsize.py`** вЂ” "Spotsize": load an Excel/text beam-scan file, select two spans on the derivative curve, Gaussian-fit the edge, compute the 95.4% width and beam diameter (О”x Г— sin(angle)).
- **`sem.py`** вЂ” "SEM": nanowire width/height/angle measurement from SEM TIFF images, a Python port of the "smarter SEM" MATLAB toolset (`\\nas.ads.mwn.de\tuze\wsi\e24\SQN\Researchers\Haubmann Benjamin\01_PhD\10_Scripts\smarter SEM`). Three modes: **Automated** (whole-image binarize в†’ contour-trace в†’ `cv2.minAreaRect` fit, with edge/size/tilt rejection rules and a per-image "skip IDs" list, port of `edgeDetect.m`), **Semi-automated** (click near a wire в†’ auto-crop a physical-size window around it в†’ detect the tallest accepted blob in the crop, falling back to a manual 2-click measurement if detection fails or the result is too short, port of `edgeDetect_semiauto.m`), and **Manual** (plain 2-click distance measurement tagged with a measurement-type label, port of `click_on_image.m`/`smarterSEM.m`). All three modes share one results table (exportable to CSV) and a live meanВ±std stats readout (replacing the separate `getValues.m`/`combined.m` post-processing scripts). Core image-processing logic lives in `sem_utils.py` (root, no PyQt imports); pixel scale is parsed by regex-scanning the TIFF's embedded ASCII metadata footer for `Pixel Size = <value> <unit>` (editable per-image if not found or wrong). Adds `opencv-python` and `Pillow` as dependencies.
- **`plot.py`** вЂ” "Plot": load one or more HDF5 spectrum files, preview each as PL Intensity (arb. units) vs. energy (one line per power step, colored darkв†’light by ascending pump fluence via matplotlib's `plasma` colormap, legend outside the axes titled "Pump Fluence (uJ/cmВІ)" with unit-less values converted from the file's mJ/cmВІ to ВµJ/cmВІ вЂ” falls back to power in mW, with its own "mW" suffix per entry and a "Power (no fluence in file)" title, if a file has no `Pump_fluence` dataset), then "Save all as PNG" writes one `.png` per loaded file next to its source `.h5` (same basename). **Deliberate exception to the pyqtgraph-only rule below**: this tab embeds a real `matplotlib.figure.Figure` via `FigureCanvasQTAgg` instead of `PGCanvas`, because the saved image must be a literal matplotlib figure (WYSIWYG with the preview) вЂ” figure construction lives in `plot_utils.py` (root, no Qt/pyplot imports, uses the `Figure` OO API directly so `savefig` works for any format regardless of the attached interactive canvas).

## Shared modules
- **`io_utils.py`** вЂ” pure data I/O: `.origin`/HDF5 read-write, power-calibration parsing, HDF5 spectrum reading (`_read_h5_spectrum`, used by the Plot tab), HDF5 structure introspection (`_h5_contents_summary`, used by the Visualizer tab's "Show Data" dialog). No UI, no plotting.
- **`sem_utils.py`** вЂ” pure image-processing logic for the SEM tab: TIFF+scale reading, blob detection/rectangle-fit, crop, and 2-point measurement. No UI.
- **`plot_utils.py`** вЂ” pure matplotlib figure-construction logic for the Plot tab (`build_spectrum_figure`, `png_path_for`). No Qt imports; `io_utils._read_h5_spectrum()` supplies the data it plots.
- **`nw_analysis.py`** вЂ” Python port of the MATLAB nanowire PL analysis pipeline (`main_Benni.m`/`Analysis.m`/`Nanowire.m`). Used by `analysis.py` as `import nw_analysis as nwa` for `fit_nw`/`integrate_spectra` on a `SimpleNamespace` "nanowire" object (`_new_nanowire()`). Moved into this repo from a sibling `..\Plot\` folder (2026-07) вЂ” previously reached via `sys.path.insert`, now just a root-level module like `io_utils.py`/`sem_utils.py`. Still imports `matplotlib.pyplot`/`matplotlib.widgets` at module level for its own standalone interactive functions (`set_startconditions`, `thresholds`), but `analysis.py` always calls `fit_nw`/`integrate_spectra` with `show_progress=False`, so those matplotlib windows are never triggered from this app вЂ” matplotlib is a hard import-time dependency of this module even though it's otherwise out of scope for the app's own plotting (see below).
- **`pl.py`** вЂ” numeric helpers: origin-file parsing, spectrum stitching (`_stitch_counts`), Gaussian fitting, `_HC_EV_NM` (eVв†”nm constant), used by `io_utils.py`, `stitch.py`, `visualizer.py`, `analysis.py`, `spotsize.py`. Copied into this repo (2026-07) from `C:\Users\ge23jud\PL Helper\pl.py`, which is its own separately tracked git repo (`github.com/ge23jud/PL-Helper.git`) вЂ” deliberately a **copy**, not a move, so that other repo's history/remote stays intact and other tools depending on it are unaffected; this repo's copy is now the one all local imports resolve to. If `pl.py` gains features upstream in `PL Helper`, re-sync manually.

## Plotting architecture вЂ” `plotting/` package
Every tab except `plot.py` plots exclusively through PyQtGraph via the shared `plotting/` package (migrated off Matplotlib in 2026-07). Each such tab imports only from `plotting` (never a submodule directly). `plot.py` is the one deliberate exception вЂ” see its entry above вЂ” and embeds matplotlib directly via `FigureCanvasQTAgg`, not `PGCanvas`. Note `matplotlib` came back as a `requirements.txt` dependency in 2026-07 when `nw_analysis.py` was moved into this repo вЂ” originally needed only for that module's own (unused-from-this-app) interactive functions and `pl.py`'s, `plot.py`'s embedded preview is the first tab to actually render matplotlib on-screen.

- **`canvas.py`** вЂ” `PGCanvas(pg.GraphicsLayoutWidget)`: the shared embeddable canvas. `reset_axes()` clears curves/ROIs/legend/colorbar and returns a fresh `plot_item` (full clear-and-rebuild on every redraw, matching every tab's existing redraw pattern вЂ” no incremental updates). `sigDataClicked(x, y, button)` replaces matplotlib's `button_press_event` for click-driven picking (used by `analysis.py`). `draw_idle()` is a no-op kept only so old call sites didn't need touching.
- **`theme.py`** вЂ” `apply_pg_theme()` (call once at startup, before any `PGCanvas`), `TAB10` palette, `PLASMA` colormap вЂ” matched to the app's dark `qt_material` theme.
- **`multiline.py`** вЂ” `MultiLinePlotter` + `LineColorScheme` strategy classes (`CategoricalScheme` = tab10-by-index, `SequentialScheme` = plasma-by-value, `LogAlphaRamp` = log-scaled opacity wrapper) вЂ” covers every "color N lines by file or by power" pattern across `stitch.py`/`visualizer.py`/`analysis.py`.
- **`colorbar.py`** вЂ” `GradientLegend(pg.ColorBarItem)`: a standalone plasma gradient legend (not attached to an image), used wherever lines are colored by power.
- **`regions.py`** вЂ” `DraggableSpan` (wraps `pg.LinearRegionItem`; one class serves both the interactive picker role and the "replay a stored span" role via `set_range()`) and `DraggableRect` (wraps `pg.RectROI`; a persistent resizable box вЂ” deliberately different UX from matplotlib's click-drag `RectangleSelector`, confirmed acceptable). Both track their own add/remove lifecycle since `reset_axes()` wipes them on every redraw.
- **`scatter.py`** вЂ” `RecolorableScatter`: thin wrapper over `pg.ScatterPlotItem` for per-point recoloring (analysis.py's threshold-region highlighting).
- **`annotations.py`** вЂ” `FilledRegion` (`pg.FillBetweenItem` wrapper) and `MeasurementArrow` (hand-built double-headed arrow + label composite вЂ” no built-in PyQtGraph equivalent; spotsize.py's only consumer).
- **`toolbar.py`** вЂ” `make_pg_toolbar(canvas)`: a minimal 2-button toolbar (Export ImageвЂ¦, Reset View) replacing matplotlib's `NavigationToolbar2QT`. Pan/zoom is handled by PyQtGraph's built-in mouse interactions (scroll-zoom, drag-pan, right-drag box-zoom) instead of dedicated buttons.

**Known pyqtgraph gotcha (worked around in `canvas.py`)**: toggling `setLogMode` right after `reset_axes()` can hit a `10**range` overflow if the ViewBox still holds a stale large linear range from whatever was plotted before the clear. `reset_axes()` explicitly resets the view to a small known range before re-enabling auto-range to avoid this.

**Known pyqtgraph limitation**: `pg.LegendItem` only renders items with an `.opts` attribute (`PlotDataItem`-like) вЂ” `LinearRegionItem`/`InfiniteLine` cannot be added via `legend.addItem(...)`. Span regions are identified by on-canvas color + sidebar labels instead of a legend entry; axvline-style markers use `InfiniteLine`'s own `label=`/`labelOpts=` instead.
